REDES DE COMPUTADORES I

Projeto 1: Servidor Web Concorrente

Desenvolvimento do Projeto

O projeto deverá ser desenvolvido em grupos de no máximo 2 integrantes.
Em todos os laboratórios o Sistema Operacional Linux encontra-se instalado. Faça uso de qualquer um dos laboratórios para a realização dos projetos.

Normas de conduta

A interação entre os grupos é estimulada, no entanto qualquer tentativa de plágio de trabalhos será punida com a nota -Nmax nos trabalhos para todos os envolvidos.

Descrição do Projeto

O objetivo deste projeto é implementar um servidor Web concorrente, capaz de processar múltiplas requisições simultâneas em paralelo. Você deverá demonstrar que seu servidor Web é capaz de enviar uma home page a um browser Web, contendo arquivos em formato html e imagens em formato jpg.

O servidor Web implementado deve utilizar a versão 1.0 do protocolo HTTP, definida na RFC 1945, de forma que requisições HTTP separadas sejam criadas para cada objeto web solicitado.

O servidor deverá ser capaz de manipular múltiplas requisições simultâneas em paralelo, de forma concorrente. O servidor deve aguardar por conexões em uma porta fixa, passada como argumento no momento de sua execução, e quando receber uma requisição de conexão TCP, ele deve atender a essa requisição em um processo ou thread separados.

Para simplificar o processo de desenvolvimento e testes, sugere-se desenvolver o código em duas etapas. Na primeira etapa, você deve escrever um servidor concorrente que simplesmente exibe o conteúdo da mensagem de requisição HTTP recebida, separando e interpretando cada um dos parâmetros enviados na requisição. Assim que esse programa funcionar adequadamente, você deve adicionar o código necessário para gerar a resposta apropriada, de acordo com a especificação do protocolo HTTP.

As requisições para o seu servidor devem ser geradas a partir de um browser Web. Lembre-se que o seu servidor provavelmente não atenderá requisições na porta padrão 80, logo, é preciso especificar o número da porta junto à URL que você fornecer ao browser Web. Por exemplo, se o nome da sua máquina é host.someschool.edu, seu servidor está escutando na porta 6789 e você quer acessar o arquivo index.html, então você deve digitar no browser a seguinte URL:

http://host.someschool.edu:6789/index.html

Se você omitir “:6789”, o browser irá assumir a porta 80, onde, provavelmente, não haverá nenhum servidor agurdando por conexões.

Quando o servidor encontrar um erro, ele deve enviar uma mensagem de resposta com o código HTTP apropriado e com um arquivo HTML descrevendo o erro, se necessário, de forma que a informação de erro seja exibida na janela do browser corretamente.
